Обзор системы MA Furniture

Интернет-магазин мебели с админ-панелью на Flask + SQLite. Система поддерживает управление товарами, разделами, фоновым изображением, резервное копирование и онлайн-трекер.

Технологический стек

  • Backend: Flask (Python 3.11)
  • База данных: SQLite
  • Frontend: HTML5, CSS3, JavaScript (ES6)
  • Стили: FontAwesome 6, кастомный CSS
  • HTTP-сервер: Gunicorn (на Amvera)

API Endpoints

Товары

  • GET /api/products - получить список товаров (с фильтрацией)
  • GET /api/products/<id> - получить товар по ID
  • GET /api/admin/products - все товары для админки
  • POST /api/admin/products - создать товар
  • PUT /api/admin/products/<id> - обновить товар
  • DELETE /api/admin/products/<id> - удалить товар

Разделы

  • GET /api/sections - все разделы
  • POST /api/admin/sections - создать раздел
  • PUT /api/admin/sections/<id> - обновить раздел
  • DELETE /api/admin/sections/<id> - удалить раздел
  • POST /api/admin/sections/reorder - изменить порядок разделов

Медиа

  • GET /api/media/background - получить фон главной страницы
  • POST /api/admin/media/background - создать/обновить фон
  • DELETE /api/admin/media/background/<id> - удалить фон
  • POST /api/upload - загрузить изображение
  • POST /api/upload/delete - удалить изображение

Заказы

  • POST /api/orders - создать заказ (отправка в Telegram)

Бэкапы

  • GET /api/admin/backup/download - скачать бэкап (БД + изображения)
  • POST /api/admin/backup/upload - восстановить из бэкапа
  • GET /api/admin/backup/list - список бэкапов
  • POST /api/admin/backup/delete - удалить бэкап

Системные

  • POST /api/admin/login - авторизация
  • GET /api/admin/verify - проверка токена
  • GET /api/admin/online/count - количество онлайн-пользователей
  • GET /api/admin/maintenance/status - статус режима обслуживания
  • POST /api/admin/maintenance/enable - включить режим обслуживания
  • POST /api/admin/maintenance/disable - выключить режим обслуживания

Админ-панель

Страницы админки находятся в папке static/admin/.

  • admin-login.html – страница входа
  • admin-dashboard.html – главная панель с виджетами
  • products-management.html – управление товарами (CRUD, цветовые копии)
  • categories-management.html – управление разделами (drag-and-drop)
  • media-management.html – загрузка/удаление фона
  • backup-management.html – резервное копирование и восстановление
  • techdocks.html – техническая документация (текущая страница)

Авторизация

Логин и пароль задаются через переменные окружения STAD и SUTT. Если они не заданы, используются значения по умолчанию: obratites / koperatoru.

Цветовые копии товаров

Для каждого товара можно создать до 4 цветовых копий. Каждая копия имеет свою цену, количество на складе и отдельные изображения.

Резервное копирование

Система создает ZIP-архив, содержащий:

  • Файл базы данных ma_furniture.db
  • Папку uploads/products/ со всеми изображениями товаров и фона

Архивы сохраняются в папку data/backups/. Восстановление заменяет текущую БД и изображения на те, что в архиве. Перед восстановлением автоматически создается резервная копия текущего состояния (с расширением .backup).

Развертывание на Amvera

Конфигурация задаётся в файле amvera.yml (менять нельзя по требованию заказчика). Основные параметры:

  • version: null
  • environment: python
  • toolchain: pip, version: "3.11"
  • build.requirementsPath: requirements.txt
  • run.scriptName: app.py
  • run.command: gunicorn --bind 0.0.0.0:5000 --workers 2 --timeout 120 wsgi:app
  • run.persistenceMount: /data – сохраняет данные между перезапусками

Перед заливкой на Amvera убедитесь, что в requirements.txt есть все зависимости: flask, flask-cors, requests, gunicorn и т.д.

Устранение неполадок

Не загружаются изображения

  • Проверьте права на папку static/uploads/products/ (должна быть доступна для записи).
  • Убедитесь, что файл не превышает 5MB.
  • Проверьте путь: загруженные изображения доступны по URL /static/uploads/products/имя_файла.

Не отправляются заказы в Telegram

  • Проверьте переменные окружения TELEGRAM_BOT_TOKEN и TELEGRAM_CHAT_ID в настройках Amvera.
  • Убедитесь, что бот добавлен в чат и имеет права на отправку сообщений.

Не работает drag-and-drop в разделе "Разделы"

  • Убедитесь, что подключена библиотека SortableJS: <script src="https://cdn.jsdelivr.net/npm/sortablejs@latest/Sortable.min.js"></script>.
  • Проверьте консоль браузера на ошибки JavaScript.

После восстановления бэкапа пропали изображения

  • Бэкап должен содержать папку uploads/products/. Если её нет, изображения не будут восстановлены.
  • Попробуйте создать новый бэкап и восстановить его заново.
