Обзор системы MA Furniture

Интернет-магазин мебели с админ-панелью на Flask + SQLite. Система поддерживает управление товарами, разделами, фоновым изображением, резервное копирование, онлайн-трекер, цветовые варианты товаров и отправку заказов в Telegram.

Технологический стек

  • Backend: Flask (Python 3.11)
  • База данных: SQLite
  • Frontend: HTML5, CSS3, JavaScript (ES6)
  • Стили: FontAwesome 6, кастомный CSS
  • HTTP-сервер: Gunicorn (на Amvera)
  • Система контроля версий: Git (рекомендуется)

Все статические файлы размещаются в папке static/, а база данных и пользовательские файлы — в data/.

Архитектура и технологии

Система построена по классической клиент-серверной схеме:

  • Frontend: статические HTML-страницы, динамически загружающие данные через JavaScript (fetch). Основные файлы: index.html, shop.html, piece.html и их соответствующие JS-скрипты.
  • Backend: Flask-приложение (app.py) предоставляет REST API. База данных SQLite инициализируется через database.py. Все данные хранятся в папке data/.
  • Взаимодействие: Клиентские скрипты (например, data-manager.js) обращаются к API, получают данные и отображают их на страницах.
  • Админ-панель: отдельный раздел с авторизацией, расположенный в static/admin/.
  • Развёртывание: На хостинге Amvera используется Gunicorn, конфигурация задана в amvera.yml. Данные сохраняются в персистентной директории /data.

API Endpoints

Товары

  • GET /api/products – получить список товаров (фильтры: ?status=active, ?section=pantographs, ?search=текст)
  • GET /api/products/<id> – получить товар по ID
  • GET /api/admin/products – все товары для админки (требуется авторизация)
  • POST /api/admin/products – создать товар
  • PUT /api/admin/products/<id> – обновить товар
  • DELETE /api/admin/products/<id> – удалить товар

Разделы (категории)

  • GET /api/sections – все разделы
  • POST /api/admin/sections – создать раздел
  • PUT /api/admin/sections/<id> – обновить раздел
  • DELETE /api/admin/sections/<id> – удалить раздел
  • POST /api/admin/sections/reorder – изменить порядок разделов (принимает {"order": [id1, id2, ...]})

Медиа (фон главной страницы)

  • GET /api/media/background – получить фон (для публичного доступа)
  • GET /api/admin/media/background – получить фон для админки
  • POST /api/admin/media/background – создать/обновить фон
  • PUT /api/admin/media/background/<id> – обновить фон
  • DELETE /api/admin/media/background/<id> – удалить фон
  • POST /api/upload – загрузить изображение (принимает file)
  • POST /api/upload/delete – удалить загруженное изображение

Заказы

  • POST /api/orders – создать заказ (требуется Telegram бот)

Цветовые варианты

  • GET /api/products/<id>/colors – получить цветовые варианты товара
  • POST /api/admin/products/<id>/color-variant – добавить цветовой вариант (требуется авторизация)

Системные

  • POST /api/admin/login – авторизация
  • GET /api/admin/verify – проверка токена
  • GET /api/admin/online/count – количество онлайн-пользователей (за последние 5 минут)
  • GET /api/admin/maintenance/status – статус режима обслуживания
  • POST /api/admin/maintenance/enable – включить режим обслуживания
  • POST /api/admin/maintenance/disable – выключить режим обслуживания
  • GET /api/admin/dashboard/stats – статистика для дашборда
  • GET /api/admin/dashboard/activity – последняя активность
  • GET /api/admin/dashboard/popular-products – популярные товары

Структура базы данных

Файл: data/ma_furniture.db. Схема создаётся автоматически при первом запуске.

Миграция из JSON

При запуске приложения скрипт database.py проверяет наличие старых JSON-файлов в data/products/, data/sections.json, data/background.json и переносит их в SQLite, если таблицы пусты. После миграции JSON-файлы остаются, но не используются.

Админ-панель

Страницы админки находятся в static/admin/.

  • admin-login.html – страница входа. Логин и пароль задаются через переменные окружения STAD и SUTT. Если они не заданы, используются значения по умолчанию: obratites / koperatoru.
  • admin-dashboard.html – главная панель с виджетами (статистика, активность, популярные товары, онлайн-счётчик).
  • products-management.html – управление товарами (CRUD, цветовые копии). Поддерживает фильтрацию, поиск, пагинацию.
  • categories-management.html – управление разделами (drag-and-drop для порядка, включение/выключение).
  • media-management.html – загрузка/удаление фона главной страницы.
  • backup-management.html – резервное копирование и восстановление (создание и загрузка ZIP-архива с БД и изображениями).
  • techdocks.html – техническая документация (текущая страница).

Авторизация и безопасность

После успешного входа токен сохраняется в localStorage и передаётся в заголовке Authorization: Bearer <token> для всех защищённых эндпоинтов. Проверка токена выполняется на сервере через /api/admin/verify.

Корзина и заказы

Корзина реализована на клиентской стороне с использованием localStorage (файл cart.js). Товар добавляется через кнопку "В корзину", количество изменяется в боковом окне. При оформлении заказа открывается модальное окно с формой. После отправки данные передаются на /api/orders.

Обработка заказа

  • Сервер сохраняет заказ в таблицу orders (SQLite).
  • Формируется сообщение для Telegram-бота и отправляется через API Telegram (если настроены переменные TELEGRAM_BOT_TOKEN и TELEGRAM_CHAT_ID).
  • После успешного создания заказа пользователь перенаправляется на страницу order-success.html с параметрами заказа.
  • Корзина очищается (удаляется запись из localStorage).

Поля формы заказа

  • ФИО (обязательно)
  • Телефон (обязательно)
  • Email
  • Адрес доставки
  • Комментарий к заказу

Если Telegram не настроен, заказ всё равно сохранится, но вернётся сообщение об ошибке отправки в Telegram.

Цветовые копии товаров

Система позволяет создавать до 4 цветовых вариантов для каждого товара. Каждый вариант имеет свой код, название цвета, HEX-код, цену, количество на складе и отдельные изображения.

  • Варианты хранятся в JSON-поле color_variants таблицы products.
  • На странице товара (piece.html) отображается круговой выбор цвета (ColorPicker). При выборе цвета обновляются цена, наличие и изображения.
  • В админ-панели для товара есть кнопка "Создать цветовую копию". Открывается модальное окно с цветовым пикером, полями цены, количества и загрузкой изображений.
  • При создании копии генерируется уникальный код вида OSNOVNOY_KOD/N, где N – порядковый номер копии.

Цветовой пикер в админке использует круговую палитру или ручной ввод HEX. Если библиотека AdminColorPicker не загружена, используется запасной вариант.

Резервное копирование

Страница управления бэкапами (backup-management.html) предоставляет возможности:

  • Экспорт: создание ZIP-архива, содержащего файл базы данных ma_furniture.db и все изображения из папки static/uploads/products/. Архив скачивается на компьютер и одновременно сохраняется в папку data/backups/.
  • Импорт: загрузка ранее созданного ZIP-архива. Перед восстановлением создаются резервные копии текущей БД и папки uploads (с расширением .backup). После успешной распаковки данные заменяются.
  • Список сохранённых бэкапов: отображает все ранее созданные архивы с возможностью удаления.

Важно: при восстановлении заменяется не только БД, но и все изображения товаров. Будьте внимательны. Рекомендуется регулярно создавать бэкапы перед крупными изменениями.

Развертывание на Amvera

Конфигурация задаётся в файле amvera.yml (менять его нельзя по требованию заказчика). Основные параметры:

version: null
meta:
  environment: python
  toolchain:
    name: pip
    version: "3.11"
build:
  requirementsPath: requirements.txt
  useCache: true
run:
  scriptName: app.py
  command: "gunicorn --bind 0.0.0.0:5000 --workers 2 --timeout 120 wsgi:app"
  persistenceMount: /data
  containerPort: 5000

Перед заливкой убедитесь, что в requirements.txt есть все зависимости: flask, flask-cors, requests, gunicorn и др.

Важно: папка static/uploads должна быть доступна для записи. На Amvera это обеспечивается монтированием /data. Файлы, сохранённые в /data, сохраняются между перезапусками. База данных и изображения, загруженные через админку, по умолчанию сохраняются в /data/.../ (см. код app.py).

Режим обслуживания

Администратор может временно отключить сайт для посетителей, включив режим обслуживания. При этом:

  • Все публичные маршруты (кроме статики, админки и API-эндпоинтов, начинающихся с /api/admin/) возвращают страницу maintenance.html с кодом 503.
  • Режим включается/выключается через кнопку в дашборде админки (эндпоинты /api/admin/maintenance/enable и /api/admin/maintenance/disable).
  • Состояние сохраняется в файл data/.maintenance.

Страница maintenance.html содержит игру "Угадай число", чтобы посетители могли скоротать время.

Устранение неполадок

Не загружаются товары на главной или в каталоге

  • Проверьте, работает ли API: откройте /api/products в браузере. Должен вернуться JSON с товарами.
  • Убедитесь, что в админ-панели есть активные товары (статус "active").
  • Проверьте консоль браузера на ошибки JavaScript (например, 404 на файлы).

Не загружаются изображения

  • Проверьте права на папку static/uploads/products/. На сервере она должна быть доступна для записи.
  • Убедитесь, что пути к изображениям в базе данных начинаются с /static/uploads/products/... и файлы физически существуют.
  • Если изображения не отображаются на странице товара, проверьте поле images в JSON товара.

Не отправляются заказы в Telegram

  • Убедитесь, что в переменных окружения Amvera заданы TELEGRAM_BOT_TOKEN и TELEGRAM_CHAT_ID. Их можно проверить в настройках проекта.
  • Проверьте, что бот добавлен в чат и имеет права на отправку сообщений.
  • При тестировании смотрите логи сервера (через Amvera: Logs).

Проблемы с резервным копированием

  • Если при скачивании бэкапа возникает ошибка, проверьте, что папка data/backups существует и доступна для записи.
  • При восстановлении бэкапа убедитесь, что архив содержит файл ma_furniture.db. Если он не содержит папку uploads/products, изображения не будут восстановлены, но БД обновится.
  • После восстановления может потребоваться очистить кэш браузера, чтобы изменения отобразились.

Цветовые варианты не отображаются на странице товара

  • Убедитесь, что в админ-панели добавлены цветовые копии (кнопка "Создать цветовую копию").
  • Проверьте, что JS-класс ColorPicker инициализирован и получил данные по эндпоинту /api/products/<id>/colors.
  • Если товар имеет только один вариант (оригинал), блок выбора цвета скрывается.

Режим обслуживания не включается

  • Проверьте, что вы авторизованы в админке. Токен передаётся в заголовках.
  • Посмотрите, создаётся ли файл data/.maintenance при включении. На сервере должны быть права на запись в папку data.
